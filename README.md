# Platformer-Game
2D Platformer Game Built in Unity 2022.3.62

Hello, and welcome to this game demonstration known as Rogue Won.
This was a weekend solo project built in 2-3 days, and was designed to showcase my ability to design and implement a game demo with a provided sprite sheet
These were the requirements:
 * Objective: Players navigate a character through a series of simple, tile-based rooms or levels with obstacles and collectables. 
 * Once the timer runs out, the run is over. Score can be determined based on how many collectables were received.

    -Perspective – Game can be 2D or 3D, your choice. Platformer, Rogue-like, whatever, your choice!

    -Simple Collectables – Have a few items scattered around for players to collect within a time limit.

    -Single Interaction Mechanic – A button or switch that opens a door or removes an obstacle.

    -Enemies/Obstacles – Adds an element of timing and avoidance.

    -Art: You must use the tilesheet that was sent to you for the visual artwork. 
     Make a palette out of it and use it however you want, be as creative as you would like!

My game plays as follows:

    -You play as a rogue that has snuck into a castle dungeon in hopes of stealing some gold.
    -You traverse through the dungeon areas collecting coins and avoiding hazards
    -You must race against the clock to collect as many coins as possible, or finding the final treasure chest
    -Game controls guide can be found in the Escape menu '?' icon
    -Enjoy the experience and see how many coins you can collect within the time limit!


Project Design:
    -For this project I took a very straight forward approach for easy understanding and quicker implementation
    -There is plenty that can be expanded on, and optimized, but I opted for overall complete gameplay experience.
    -The game is build almost entirely with the sprite sheet provided, aside from a character model and some coins.
    -For future additions, I thought of adding some enemies to attack the player and increase difficulty
    -Additionally, maybe support mobile or WebGL for browser based gameplay
